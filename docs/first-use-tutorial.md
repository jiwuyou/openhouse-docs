# 首次使用教学

本文档定义 OpenHouse 首次教学的产品脚本。当前教学以 Android 原生桌面为入口，不再使用旧的侧栏逐项介绍流程。

## 教学目标

完成教学后，用户应知道：

- 从 OpenHouse 桌面点击图标进入应用。
- 在应用顶部点击“控制”，进入该应用自己的运行控制页。
- 由用户亲自关闭和启动应用服务，并等待真实状态确认。
- 从运行控制返回原应用，再点击“桌面”退出应用。
- 终端教学仍是独立入口，不属于开局教学。

## 教学目标应用

教学默认优先选择 `pi-agent`，但目标必须同时满足：

- 当前显示在桌面且可以打开。
- 可以在 OpenHouse 页面内打开。
- 声明了运行控制入口。
- 能在当前 service-manager 服务列表中解析到真实注册服务。
- 教学服务不是 `service-manager` 本身。

如果 `pi-agent` 不满足条件，则选择第一个符合条件的桌面应用。没有可用目标时，界面明确提示应用尚未准备好，不伪造教学完成。

## 通用交互规则

- 打开应用、关闭服务、启动服务和返回桌面都必须由用户真实点击。
- 教学不得自动执行有副作用的启停动作。
- 服务动作提交后，必须读取 service-manager 的真实状态；只收到成功响应不算完成。
- 需要用户点击的步骤在 20 秒后才允许跳过。
- 用户从控制页直接返回时，教学保持未完成，并提供重新进入控制页的入口。

## 教学步骤

### 1. 认识桌面

- 界面：OpenHouse 原生桌面。
- 操作：说明桌面是日常应用入口。
- 完成条件：用户点击“下一步”。

### 2. 从桌面进入应用

- 箭头指向：选定应用的桌面图标。
- 操作：用户真实点击图标。
- 完成条件：应用在 OpenHouse 页面内打开。
- 跨分页时，教学先自动切换到目标图标所在页，再显示箭头。

### 3. 打开单应用控制

- 箭头指向：应用顶部的“控制”。
- 操作：用户真实点击。
- 完成条件：进入该应用的控制页，并固定本次教学使用的 component ID 和 service ID。

### 4. 完成真实启停闭环

服务当前运行时：

1. 用户点击“关闭”。
2. 等待 service-manager 确认 stopped。
3. 用户点击“启动”。
4. 等待 service-manager 确认 running。

服务初始已停止时：

1. 用户先点击“启动”，等待确认 running。
2. 用户点击“关闭”，等待确认 stopped。
3. 用户再次点击“启动”，等待确认 running。

只有已经确认过停止，并确认最终恢复运行，才算完成运行控制教学。

### 5. 返回应用

- 箭头指向：“返回应用”。
- 操作：用户真实点击。
- 完成条件：控制页以成功结果返回原来的应用页面。

普通返回键、跳过或中途退出不会产生成功结果，也不会把教学误标为完成。

### 6. 从应用退出到桌面

- 箭头指向：应用顶部的“桌面”。
- 操作：用户真实点击。
- 完成条件：返回 OpenHouse 桌面并清除教学阶段状态。

## 教学入口

教学入口保留在：

- 首次安装完成页。
- OpenHouse 菜单中的“使用教学”。
- 使用教学页面。

终端教学继续使用单独入口“终端教学”。

## 不进入首次教学的内容

- Termux 命令和 Ubuntu 路径。
- service-manager REST API。
- 日志文件路径。
- 模型、Codex、Claude Code 和 CloudCLI 配置。
- Shizuku 进阶控制。
- 桌面编辑、图标拖动和高级设置。

这些内容由独立教学或 AI 参考文档说明。
