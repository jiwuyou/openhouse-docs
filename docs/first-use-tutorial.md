# 首次使用教学

本文档定义 OpenHouse 首次教学的产品脚本。首次教学只帮助普通用户开始使用，不进入终端教学。

## 教学目标

首次教学完成后，用户应知道：

- 菜单在哪里。
- `pi-agent` 是首次配置助手。
- `cc/codex` 是 Claude Code / Codex 相关入口。
- App 前台会自动保持核心服务运行。
- 一般不需要使用终端。
- 需要终端时可以单独打开终端教学。
- 可以重新打开使用教学。
- 可以点击“停止运行栈”暂时停止 OpenHouse 服务、保留当前界面并暂停本次会话自动保活；也可以点击“全部退出 OpenHouse”关闭 OpenHouse 界面并请求关闭 Termux 前台服务和终端会话。

## 通用交互规则

每一步必须定义：

- 界面。
- 箭头指向。
- 点击主体：用户点击、系统模拟、只显示下一步。
- 是否允许跳过。
- 完成条件。

跳过规则：

- 需要用户真实点击的步骤，首次 20 秒内不允许跳过。
- 不需要用户点击的说明步骤，只显示“下一步”。
- 用户主动重新打开教学时，可以允许立即跳过已完成步骤。

## 教学入口

教学入口应至少出现在：

- 首次安装完成页。
- 主菜单。
- 设置页或帮助页。

入口名称建议：

```text
使用教学
```

终端教学是单独入口，名称建议：

```text
终端教学
```

首次教学中只说明“终端一般不需要使用，需要时可以单独查看终端教学”，不带用户进入终端。

## 教学步骤

### 1. 认识菜单

- 界面：主菜单或侧边栏。
- 箭头指向：菜单按钮或侧边栏入口。
- 点击主体：用户点击。
- 是否允许跳过：20 秒后允许。
- 文案要点：这里是 OpenHouse 的主要入口。你可以从这里进入 SmallPhone、pi-agent 和 cc/codex。
- 完成条件：菜单已打开。

### 2. 认识大服务入口

- 界面：打开后的菜单。
- 箭头指向：`SmallPhone`、`pi-agent`、`cc/codex` 三个入口。
- 点击主体：只显示下一步。
- 是否允许跳过：不需要跳过按钮，只显示下一步。
- 文案要点：这些是常用的大服务。普通使用先记住 pi-agent 和 cc/codex。
- 完成条件：用户看到三个入口。

### 3. 进入 pi-agent

- 界面：菜单。
- 箭头指向：`pi-agent`。
- 点击主体：用户点击。
- 是否允许跳过：20 秒后允许。
- 文案要点：pi-agent 是首次配置助手，它会帮你阅读系统文档、配置模型，并引导 Claude 或 Codex 可用。
- 完成条件：pi-agent 页面打开。

### 4. 说明模型配置

- 界面：pi-agent 页面或模型配置提示区域。
- 箭头指向：模型配置入口或对话框。
- 点击主体：只显示下一步，实际配置由 pi-agent 引导。
- 是否允许跳过：不需要跳过按钮，只显示下一步。
- 文案要点：你只需要准备 `base_url`、`key/token`、`model id`。不需要理解所有配置文件。
- 完成条件：用户理解后点击下一步。

### 5. 认识 cc/codex

- 界面：菜单。
- 箭头指向：`cc/codex`。
- 点击主体：用户点击。
- 是否允许跳过：20 秒后允许。
- 文案要点：这里是 Claude Code、Codex 和 CloudCLI 相关入口。配置完成后可以从这里使用主要 AI 工具。
- 完成条件：cc/codex 页面或入口打开。

### 6. 说明核心服务自动运行

- 界面：运行状态或菜单。
- 箭头指向：状态区域。
- 点击主体：只显示下一步。
- 是否允许跳过：不需要跳过按钮，只显示下一步。
- 文案要点：App 在前台时会自动保持核心服务运行。前期你不需要学习服务启动和关闭。
- 完成条件：用户看到运行状态说明。

### 7. 说明终端后置

- 界面：菜单或帮助页。
- 箭头指向：终端教学入口。
- 点击主体：只显示下一步。
- 是否允许跳过：不需要跳过按钮，只显示下一步。
- 文案要点：一般不需要使用终端。以后需要高级排障或开发时，可以单独查看终端教学。
- 完成条件：用户知道终端教学独立存在。

### 8. 说明全部退出

- 界面：菜单、设置页或运行控制页。
- 箭头指向：`全部退出`。
- 同屏需要能看到或说明 `停止运行栈`。
- 点击主体：只显示下一步，不在首次教学中真的退出。
- 是否允许跳过：不需要跳过按钮，只显示下一步。
- 文案要点：如果只是暂时省资源，可以点击“停止运行栈”，它会停止 OpenHouse 运行栈、保留当前界面，并暂停本次会话自动保活；如果这次完全不用了，可以点击“全部退出 OpenHouse”，它会停止 OpenHouse 运行栈、关闭 OpenHouse 界面，并请求关闭 Termux 前台服务和终端会话。再次打开 App 会重新启动核心服务，除非你在高级设置里关闭了自动保活。
- 完成条件：用户知道全部退出入口。

### 9. 完成教学

- 界面：教学完成页。
- 箭头指向：`开始使用`。
- 点击主体：用户点击。
- 是否允许跳过：不需要。
- 文案要点：接下来进入 pi-agent，按照它的提示配置 Claude 或 Codex。
- 完成条件：教学标记完成，进入 pi-agent 或返回菜单。

## 不进入首次教学的内容

以下内容不进入首次教学：

- Termux 命令。
- Ubuntu 路径。
- proot-distro。
- service-manager API。
- 日志文件路径。
- Hermes 安装。
- Shizuku 进阶控制。
- 文件中转站高级用法。

这些内容进入独立教学或 AI 参考文档。
